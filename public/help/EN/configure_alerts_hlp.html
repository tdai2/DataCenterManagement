Alerts - Help

When one of the selected system events occurs, an alert is generated and sent to the configured destination(s). Each LAN channel can have up to two destinations.

Globally Enable Platform Event Filtering

Global control for enabling or disabling platform event filtering. When filtering is globally disabled through this setting, alerts will not be sent. This can be used to prevent sending alerts until you have fully specified your desired alerting policies.

Log Events For Filter Actions

When a filter match is made, and an action is taken, an event is added to the system event log.

Select Events

Select one or more system events that will trigger an Alert. Clearing all events disables Alerts. These events correspond to the IPMI preconfigured Platform Event Filters.

LAN Channel

Select which LAN Channel to configure destinations for. Each LAN Channel has its own set of up to two destinations. If LAN failover mode is enabled, then only one channel will be listed for configuration

Alert Destinations

Alert destinations can be one of two types:

  • SNMP Trap
  • Email (requires Alert Email to be configured)

Actions
Check All

Check all events to generate Alerts.

Clear All

Uncheck all events so no Alerts will be generated.

Save

Click 'Save' to save any changes made.

Send Test Alert

To test whether an alert will reach its destination, set the LAN Channel field to the desired channel and configure at least one destination. Then click 'Send Test Alerts' to send a simple test alert to the destination(s) for that Channel.
