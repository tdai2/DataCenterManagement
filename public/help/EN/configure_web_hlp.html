Security - Help

Use this page to configure security settings for server management.

Failed Login Attempts

Set the number of failed login attempts a user is allowed before being locked out. Zero means no lockout. Default is 3 attempts.

User Lockout Time (min)

Set the time in minutes that the user is locked out before being allowed to login again. Zero means no lockout and unlocks all currently locked out users. Default is 1 min.

Force HTTPS

Enable this option to force the User to user Secure Login for Web using HTTPS protocol. It will use the certificate uploaded under 'Configuration->SSL'.

Web Session Timeout

Set the maximum web service timeout in seconds. Timeout should be between 60 and 10800 seconds. Default timeout is 1800 seconds.

Port Settings

Set the ports used for http (default: 80) and https (default: 443) web sessions. Changing either of these settings will immediately terminate all current web sessions.

Optional Network Services

The optional network services currently enabled. For security reasons, unused services can be disabled and the associated network ports closed.

  • - SSH is used for SMASH-CLP command line
  • - HTTP/HTTPS is used for Web Console
  • - IPMI over LAN is used for RMCP/RMCP+
